■ 접근제어 지시자의 관계
			 (같은 페이지)
지시자	  클래스 내부	 동일 패키지     상속받은 클래스    이외의 영역
private	       ○             X	               X	         X
default	       ○             ○	       X	         X

public 	       ○             ○	       ○		○


※ public > protected > default > private

※ 클래스 설계 기법
   객체 지향의 관점에서는 아주 커다란 하나의 클래스보다
   아주 작은 여러 개의 클래스가 훨씬 큰 위력을 갖는다.

※ 캡슐화는 객체지향에서 실제 프로젝트에 적용하기 쉽지 않은 개념이다.
   어디까지 묶고, 객체지향화할 것인지 정하기 나름이므로 정답이 없다.
